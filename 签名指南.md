# APK 签名指南

## 概述

Debug APK 可以直接安装测试，但 Release APK 需要签名才能安装到设备上。

## 方案一：GitHub Actions 自动签名（推荐）

### 步骤 1：创建签名密钥

在本地终端执行：

```bash
# 生成签名密钥
keytool -genkey -v \
  -keystore my-release-key.jks \
  -keyalg RSA \
  -keysize 2048 \
  -validity 10000 \
  -alias my-alias

# 按提示输入：
# - 密钥库密码（记住这个密码）
# - 姓名、组织等信息（可以随便填）
# - 密钥密码（通常和密钥库密码相同）
```

执行后会生成 `my-release-key.jks` 文件。

### 步骤 2：转换为 Base64

**macOS/Linux:**
```bash
base64 my-release-key.jks
```

**Windows PowerShell:**
```powershell
[Convert]::ToBase64String([IO.File]::ReadAllBytes("my-release-key.jks"))
```

复制输出的 Base64 字符串。

### 步骤 3：配置 GitHub Secrets

1. 打开 GitHub 仓库页面
2. 点击 **Settings** → **Secrets and variables** → **Actions**
3. 点击 **New repository secret**
4. 添加以下 Secrets：

| Secret 名称 | 值 |
|------------|-----|
| `SIGNING_KEY` | Base64 编码的 jks 文件内容 |
| `KEY_ALIAS` | 密钥别名（如 `my-alias`）|
| `KEY_STORE_PASSWORD` | 密钥库密码 |
| `KEY_PASSWORD` | 密钥密码 |

### 步骤 4：推送代码触发构建

```bash
git add .
git commit -m "Add signing configuration"
git push
```

构建完成后，在 Actions 页面可以下载 `release-apk-signed`。

## 方案二：本地签名（备用）

如果 GitHub Actions 签名失败，可以下载未签名的 Release APK 本地签名：

### 1. 下载未签名 APK

从 GitHub Actions 下载 `release-apk-unsigned.apk`

### 2. 使用 apksigner 签名

```bash
# 找到 Android SDK 中的 apksigner
# 通常在: ~/Library/Android/sdk/build-tools/34.0.0/apksigner (macOS)
# 或: %LOCALAPPDATA%\Android\Sdk\build-tools\34.0.0\apksigner.bat (Windows)

# 签名命令
apksigner sign \
  --ks my-release-key.jks \
  --ks-key-alias my-alias \
  --out app-release-signed.apk \
  app-release-unsigned.apk

# 按提示输入密码
```

### 3. 验证签名

```bash
apksigner verify app-release-signed.apk
```

### 4. 安装测试

```bash
adb install app-release-signed.apk
```

## 方案三：Android Studio 签名（最简单）

1. 下载 GitHub Actions 生成的 `release-apk-unsigned.apk`
2. 打开 Android Studio
3. **Build** → **Generate Signed Bundle / APK...**
4. 选择 **APK**
5. 点击 **Create new...** 创建密钥（或使用已有密钥）
6. 选择 Release APK 文件进行签名

## 常见问题

### Q: 签名后安装提示"应用未安装"

**A:** 可能是以下原因：
- 设备上已存在同名应用但签名不同 → 先卸载旧版本
- APK 损坏 → 重新下载并签名
- Android 版本不兼容 → 检查 minSdk

### Q: GitHub Actions 签名步骤报错

**A:** 检查 Secrets 是否正确配置：
- `SIGNING_KEY` 必须是 Base64 格式
- `KEY_ALIAS` 必须和创建时一致
- 密码不能包含特殊字符（如 `$` `"`）

### Q: 如何更新签名密钥？

**A:** 
1. 生成新的密钥
2. 更新 GitHub Secrets
3. 重新推送代码

**注意：** 更换密钥后，用户需要卸载旧版本才能安装新版本。

---

## 安全提示

1. **不要上传密钥文件到 GitHub** - 使用 Secrets 存储
2. **备份密钥文件** - 丢失后无法更新应用
3. **密钥密码要复杂** - 防止被破解
4. **妥善保管 Base64 字符串** - 它等同于密钥文件
